## Reproducing the Results

Every strategy ships as a runnable TypeScript file inside the `example/` directory of the Backtest Kit repository. To reproduce any result:

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/tripolskypetr/backtest-kit.git
    cd backtest-kit/example
    npm install
    ```
  </Step>
  <Step title="Run the backtest">
    Each example includes its exact CLI command. For example, to reproduce the SHORT DCA ladder:

    ```bash
    npm start -- --backtest --symbol BTCUSDT \
      --strategy mar_2026_strategy \
      --exchange ccxt-exchange \
      --frame mar_2026_frame \
      ./content/mar_2026.strategy/mar_2026.strategy.ts
    ```
  </Step>
  <Step title="Inspect the report">
    The framework automatically generates a Markdown performance report in `./dump/backtest/`. Open it to see trade-by-trade PnL, Sharpe ratio, drawdown, and win rate — identical to the numbers shown in this documentation.
  </Step>
  <Step title="Open the web UI (optional)">
    Add `--ui` to any command to launch the interactive dashboard at `http://localhost:60050`, where you can inspect each signal's state transitions in real time.

    ```bash
    npm start -- --backtest --symbol BTCUSDT --ui \
      ./content/mar_2026.strategy/mar_2026.strategy.ts
    ```
  </Step>
</Steps>

<Note>
  Historical candle data is fetched automatically from Binance via CCXT and cached locally on first run. Subsequent runs of the same symbol and date range use the cache, making iteration fast.
</Note>
